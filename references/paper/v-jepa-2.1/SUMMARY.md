# V-JEPA 2.1 — Apply the JEPA latent-prediction loss to *every* token (visible context + masked) plus deep multi-level self-supervision to unlock dense spatio-temporal features, while keeping (and improving) global video understanding and world-modeling

**Authors:** Lorenzo Mur-Labadia, Matthew Muckley, Amir Bar, Mido Assran, Koustuv Sinha, Mike Rabbat, Yann LeCun, Nicolas Ballas, Adrien Bardes (FAIR at Meta; Universidad de Zaragoza) **Venue/Year:** arXiv preprint, 2026 (v2, 17 Mar 2026) **arXiv:** 2603.14482 **Repo:** https://github.com/facebookresearch/vjepa2

## TL;DR
V-JEPA 2.1 fixes the one thing V-JEPA 2 is bad at: dense, spatially-grounded features. The diagnosis is that the V-JEPA loss is applied *only to masked tokens*, so visible context tokens are free to act as global "register"-like aggregators and never have to encode local spatial structure. The fix is a **Dense Prediction loss** = original masked-token L1 loss + a new **context loss** `Lctx` that also supervises the (unmasked) context tokens against the EMA target, with a distance-aware weight that emphasizes context patches near masked regions. On top of this they add **Deep Self-Supervision** (apply both losses at four encoder levels via a multi-level predictor), a **multi-modal tokenizer** (2D conv for images, 3D conv for videos, plus a learnable modality token, instead of duplicating images into fake 16-frame videos), a larger/more-diverse image+video dataset (VisionMix-163M), scaling to ViT-G (2B), and a high-res cooldown. Result: SOTA on dense linear-probe tasks (0.307 RMSE NYUv2 depth, beating DINOv3 ViT-7B) and on predictive/global video tasks (7.71 mAP Ego4D STA, 40.8 R@5 EPIC-KITCHENS anticipation, 77.7 SSv2), plus a +20% real-robot grasp success rate over V-JEPA-2 AC and 10x-faster navigation planning. All evals use a frozen encoder.

## Problem & motivation
World models need representations that simultaneously (i) preserve dense spatio-temporal structure (localization, geometry, tracking) and (ii) capture dynamics + support global recognition. V-JEPA-style models nail global video understanding and planning but, as PCA visualizations show, produce noisy, fragmented feature maps and score poorly on dense linear-probe tasks (22.2 mIoU ADE20K, 0.682 RMSE NYUv2 for V-JEPA 2). Image-only methods like DINOv2/v3 give clean dense features but do not learn temporal dynamics from video. The central empirical finding: high-quality dense features do **not** emerge when the predictive loss is applied only to masked regions. The authors hypothesize this is because the predictor outputs a token for both context and masked positions, but the loss ignores context outputs, so the model has no incentive to keep local info in context tokens and instead uses them as global aggregators (analogous to register tokens). Supervising the context tokens too should force local grounding.

## Method
Built on the V-JEPA family: encoder `Eθ` processes a masked view `x` (patches dropped); context tokens are concatenated with learnable mask tokens carrying spatio-temporal position; predictor `Pϕ` outputs an embedding for every token; targets come from an EMA target encoder `Eθ̄` over the full clean view `y`; stop-gradient on targets prevents collapse; L1 loss. The original objective `Lpredict` is L1 over masked tokens only (Eq. 1). The five ingredients:

1. **Dense Prediction loss (context loss `Lctx`).** Add an L1 term over the *context* (visible) tokens vs the EMA target (Eq. 2), minimizing `Lpredict + Lctx`. Naively this collapses global semantics (the model can trivially copy context features), so the weight is critical. A fixed `λ` trades dense gains for action-recognition loss; the chosen scheme is (a) a **warm-up** of `λ` over epochs 50–100 to stabilize, plus (b) a **distance weighting** `λi = λ / sqrt(d_min(i, M))`, where `d_min` is the block distance from context token `i` to the nearest masked token — emphasizing patches near masked regions to enforce local continuity. This is the load-bearing innovation; it alone takes ADE20K 22.2→33.9 mIoU and NYUv2 0.682→0.473 RMSE, but degrades SSv2 (72.8→62.5) and IN1K (82.2→72.6).
2. **Deep Self-Supervision.** Concatenate the outputs of three intermediate encoder blocks + the final layer along the channel axis, fuse with a lightweight MLP, feed the predictor, and have it emit four predictions (one per supervised level). Both `Lpredict` and `Lctx` are applied at all four levels. This *restores* the global performance lost by the context loss (back to ~72 SSv2 / 80.8 IN1K) while further improving dense tasks (38.6 mIoU, 0.463 RMSE), and lets dense downstream tasks use only the final-layer features (no intermediate-layer fishing needed).
3. **Multi-modal tokenizer.** 3D conv (16×16×2) for video, 2D conv (16×16) for images, plus a learnable modality token in both encoder and predictor — instead of V-JEPA 2's single 3D conv that duplicated each image into a 16-frame "static video" (wasteful + wrong inductive bias). Images and videos go to separate workers; gradients are aggregated each step; the image/video ratio is set by per-modality batch size (128 video clips of 16 frames + 2,304 images per global batch).
4. **Data scaling (VisionMix-163M).** Replace V-JEPA 2's 1M-image ImageNet subset with LVD-142M (curated), and re-weight video toward dynamic/diverse content (SSv2 0.056→0.170, YT-1B 0.188→0.720). Improves all tasks.
5. **Model scaling + cooldown.** Scale ViT-L (300M) → ViT-G (2B); then a 12k-iteration cooldown that decays LR and raises resolution (video 16f@256→64f@384, images 256→512). Cooldown especially helps depth (0.365→0.307 NYUv2). Schedule: warmup-constant LR, 135k iterations, fixed EMA + weight decay, then cooldown.

**Distillation.** ViT-G (2B) → ViT-L (300M) / ViT-B (80M): frozen teacher instead of EMA target, an EMA copy of the student kept as the final model, last-layer-only loss (no deep self-supervision), and a smaller 12-block predictor. Ablations run after phase 1; final numbers use full warmup-constant-cooldown.

## Key results
All with a **frozen encoder** (linear or attentive probes).
- **Component ablation (ViT-L, single-image ADE20K seg / SSv2 cls):** mIoU 22.2 → 33.8 (+ctx loss) → 38.6 (+deep SS) → 40.8 (+VisionMix) → 41.4 (+multimodal tok) → 47.1 (+ViT-G) → 47.9 (+cooldown). The context loss alone tanks SSv2 (72.8→62.5); deep self-supervision recovers it (→72.1). Distance-weighted + warmed-up `λ` is the best weighting scheme (Table 2).
- **Dense tasks (ViT-G, Table 8):** **0.307 RMSE NYUv2 depth — SOTA, beating DINOv3 ViT-7B (0.309)** and PEspatial-G; 2.461 KITTI; segmentation 47.9 ADE20K / 73.5 Cityscapes / 85.0 VOC12 (massive jump over V-JEPA 2: +23.4 / +27.6 / +20.7); VOS 69.0 DAVIS / 72.7 YouTube-VOS (2nd only to DINOv3). vs V-JEPA 2's 0.642 NYUv2 / 24.4 ADE20K.
- **Predictive video:** **7.71 mAP Ego4D short-term anticipation** (~+35% over prior task-specific SOTA STAformer), **40.8 R@5 EPIC-KITCHENS-100 action anticipation** (new SOTA, +2.8 over V-JEPA 2 at the 2B size).
- **Global recognition (Table 9):** **77.7 SSv2 (SOTA)**, 89.2 Diving-48, 87.7 K400, 85.5 IN1K — competitive on appearance, best on motion.
- **Robotics — manipulation (Table 6):** action-conditioned predictor (24-layer, ~300M, trained on DROID with teacher-forcing + 2-step rollout) on a real Franka arm, zero-shot, MPC via CEM. Grasp 60%→70% at horizon 1, and **80% with longer 8-step rollouts** (+20% over V-JEPA-2 AC); V-JEPA 2 *degrades* with longer horizons. Failures are gripper-timing, not spatial.
- **Robotics — navigation (Table 7):** a CDiT world model (NWM-style) on V-JEPA 2.1 features hits 5.687 ATE on TartanDrive with **10x faster planning** (10.6s vs 103.2s) than NWM/SD-VAE, using only 8 DDIM denoising steps vs ≥128; key tricks = predict the clean representation (not noise) + DDIM.
- **VideoQA:** with Llama-3.1-8B backbone, slightly better than V-JEPA 2 on matched data (83.1 PerceptionTest val); 64-frame context.
- **Distillation:** ViT-L distilled from ViT-G nearly closes the gap (SSv2 74.2→76.5 vs 77.7; ADE20K 42.0→46.7).

## Relevance to the EB-JEPA hackathon
This is a **video / world-model track informer** (`video_jepa` and `ac_video_jepa`) whose single most transferable idea is a **loss-side change**, which maps cleanly onto eb_jepa's modular encoder/predictor/regularizer split:

- **Most reusable trick → Tracks 8, 9, 10 (and any video_jepa run).** The "supervise context tokens, not just masked ones, with a distance-weighted L1 against the EMA target" idea is small, surgical, and exactly the kind of one-knob controlled comparison the guide rewards. In eb_jepa terms it is an extra term in the JEPA loss over the predictor's context-token outputs, weighted by block-distance to the nearest mask. A team could A/B masked-only vs masked+context loss on the video_jepa example and read off the dense-feature gain (e.g. via a PCA-of-patch-features visualization or a frozen linear probe), which is a self-contained 24h deliverable.
- **Track 8 (hierarchical / multi-timescale world model)** is the closest conceptual neighbor: V-JEPA 2.1's **Deep Self-Supervision** supervises at multiple *encoder depths* via a multi-level predictor. That is not multi-*timescale*, but it is the same "add supervision/predictors at more than one level and fuse" structural move, and it is the term that lets the new dense loss coexist with global understanding — a useful precedent for adding a second (coarse) predictor without breaking the base recipe.
- **Track 9 (stress-test the recipe under factors of variation).** V-JEPA 2.1 is direct evidence about *which term breaks first*: adding the context loss with too high a fixed `λ` collapses global semantics, and the fix is warm-up + distance weighting + deep supervision. That is a concrete factor-of-variation story (loss weighting vs depth/geometry robustness) and the robotics result ("better depth/dense features unlock longer planning horizons") is exactly the kind of perturbation-vs-success signal Track 9 wants.
- **Track 7 (learned planning cost).** The manipulation section uses the *same* eb_jepa-style action-conditioned predictor + CEM/MPC distance-in-latent planning; V-JEPA 2.1 shows the planner improves purely from better dense features, a useful baseline to beat with a learned cost.
- **Mechanistically it sits in the masking + EMA-target + stop-gradient (I-JEPA/V-JEPA) camp**, with L1 latent loss and no VICReg/SIGReg variance-covariance term — so the natural eb_jepa experiment is: does the context-token dense loss compose with eb_jepa's explicit covariance regularizer, or do they fight? (Note: eb_jepa author Basile Terver is a co-author on a *cited* reference, "Back to the features: DINO as a foundation for video world models," not on this paper.)

## Caveats / open threads
- **Scale is enormous and out of hackathon reach as-is:** 2B params, VisionMix-163M (142M curated images + millions of video hours), 135k+12k iterations, multi-node modality-split training. The *insight* (context loss + deep supervision) is cheap and portable; the *numbers* are not reproducible at hackathon scale — port the loss, not the recipe.
- **The context loss is delicate.** Fixed `λ` trades dense vs global performance; the paper needs a `λ` warm-up (epochs 50–100) AND inverse-sqrt distance weighting AND deep self-supervision to avoid the trivial "copy context features" collapse of global semantics. Anyone porting it must expect to tune the weight schedule, not just drop in the term.
- **Segmentation still trails image-only SOTA** on cluttered datasets (ADE20K 47.9, Cityscapes 73.5 vs DINOv3 ViT-7B's 55.9 / 81.1); authors attribute this to VisionMix having fewer cluttered scenes. Depth/VOS are the strong dense suits, not fine-grained multi-class segmentation.
- **Ablations are at ViT-L after phase 1 only;** component-by-component numbers are not all at the final ViT-G + cooldown scale, so additivity of gains at scale is assumed, not fully shown per-component.
- **No VICReg/SIGReg comparison.** Collapse (both representation and the "copy context" trivial solution) is handled by stop-grad EMA + loss weighting, never by an explicit variance-covariance regularizer — exactly the axis eb_jepa adds, and an untested interaction.
