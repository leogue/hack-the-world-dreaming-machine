# VL-JEPA — A non-generative vision-language model that predicts continuous text embeddings instead of autoregressively decoding tokens.

**Authors:** Delong Chen, Mustafa Shukor, Théo Moutakanni, Willy Chung, Jade Yu, Tejaswi Kasarla, Yejin Bang, Allen Bolourchi, Yann LeCun, Pascale Fung (Meta FAIR, HKUST, Sorbonne Université, NYU) **Venue/Year:** arXiv preprint, 2026 (v2, Feb 2026; v1 Dec 2025) **arXiv:** 2512.10942 **Repo:** none stated

## TL;DR
VL-JEPA recasts vision-language modeling as a JEPA: rather than autoregressively generating answer tokens, it predicts the *continuous embedding* of the target text and decodes to words only when needed. An X-Encoder (frozen V-JEPA 2 ViT-L) maps visual input to visual embeddings, a Predictor (8 layers of Llama-3.2-1B, query-conditioned) predicts the target text embedding, and a Y-Encoder (EmbeddingGemma-300M) provides the target embedding; a lightweight Y-Decoder reads embeddings back to text at inference. Trained with bidirectional InfoNCE in a shared 1536-d space, VL-JEPA beats a token-space VLM baseline under strictly matched conditions with ~50% fewer trainable params, beats CLIP/SigLIP2/Perception-Encoder on 8 video classification + 8 retrieval datasets, matches InstructBLIP/Qwen-VL on 4 VQA datasets at only 1.6B params, and supports selective decoding that cuts decode ops ~2.85x for streaming video.

## Problem & motivation
Standard token-generative VLMs ((X_V, X_Q) -> Y autoregressively) have two flaws the authors target. (1) Training wastes capacity modeling task-irrelevant *surface* linguistic variability (word choice, style, paraphrase): for one input, many valid answers are near-orthogonal in one-hot token space, forcing the model to fit multiple disjoint high-density modes. (2) Autoregressive token-by-token decoding adds latency and must finish before semantics are revealed, which is bad for real-time streaming video (e.g. live action tracking on smart glasses, online planning/robotics). Predicting in an abstract embedding space collapses semantically equivalent answers to nearby points (a simpler, near-unimodal target), and a single non-autoregressive forward pass yields a continuous semantic stream that can be decoded selectively.

## Method
Four components, trained on triplets <X_V (image/video), X_Q (query/prompt), Y (target text)>:
1. **X-Encoder** (X_V -> S_V): frozen V-JEPA 2 ViT-L (304M), 256px frames, video duplicated for images.
2. **Predictor** (<S_V, X_Q> -> Ŝ_Y): last 8 layers of Llama-3.2-1B (490M trainable), Llama tokenizer/embeddings for the query, **causal mask disabled** (bi-directional) so vision and query tokens jointly attend; linear projections bridge vision/text; average pool over non-[PAD] tokens gives the predicted target embedding.
3. **Y-Encoder** (Y -> S_Y): EmbeddingGemma-300M, max ctx 512, **LR multiplier x0.05** (key trick: target quality is poor early in training). Linear heads on both Predictor and Y-Encoder give a shared 1536-d space.
4. **Y-Decoder** (Ŝ_Y -> Ŷ): not used during main training; reads embeddings to text only at inference when needed.

**Objective:** L_VL-JEPA = D(Ŝ_Y, S_Y) in embedding space (vs L_VLM = D(Ŷ, Y) in token space). They use **bidirectional InfoNCE** (Predictor and Y-Encoder trained jointly, learning from each other), chosen for maturity; InfoNCE splits into an alignment term + a uniformity term that doubles as the **anti-collapse regularizer**. The paper explicitly notes VICReg / SIGReg, EMA target encoder, or freezing the Y-Encoder are valid alternative anti-collapse strategies, left to future work. **Multitasking** from one architecture: generation (caption/open VQA via decode), CLIP-style open-vocab classification + discriminative VQA (encode candidate labels, pick nearest to Ŝ_Y), text-to-video retrieval. **Selective decoding:** agglomerative clustering with temporal-connectivity constraints partitions the embedding stream into monosemantic segments (Ward/variance), decode once per segment (from exact or average-pooled embedding).

**Two-stage training:** (1) query-free pretraining on caption data (Datacomp + YFCC-100M images, Action100M video captions on HowTo100M); image-only first (1 frame, batch 24k, 100k iters -> 2B samples, 61.6% ImageNet zero-shot), then 8-frame video (60k iters) and 32-frame (10k iters); constant LR 5e-5; 4 weeks on 24 nodes x 8 H200 -> **VL-JEPA_BASE**. (2) Supervised finetuning on the PLM data mixture (25M VQA, 2.8M caption, 1.8M classification + downsampled pretrain data to fight forgetting); 83k steps, batch 3072, cosine anneal -> **VL-JEPA_SFT**.

## Key results
- **Controlled embedding-vs-token comparison** (same frozen PE ViT-L, data, batch 128, schedule; only the loss differs; 0.5B predictor vs 1B LLM): VL-JEPA's curve rises far sharper. At 15M samples seen it reaches 14.8 CIDEr / 41.0% top-5 vs the VLM's 7.1 CIDEr / 27.2% top-5 — higher sample efficiency and absolute performance with ~half the trainable params.
- **Zero-shot (VL-JEPA_BASE, ViT-L, 1.6B, only 3.3B samples seen):** 52.5 avg classification (8 datasets) vs PE-Core-G's 44.7; 63.7 avg retrieval R@1 (8 datasets) vs 58.1 — beating CLIP, SigLIP2, PE-Core despite seeing far less data (3.6B vs 86B pairs). Especially strong on motion-centric benchmarks (SSv2, EK100, EgoExo4D); weaker on appearance-centric (Kinetics-400).
- **VQA (VL-JEPA_SFT, 1.6B, discriminative):** GQA 61.5, TallyQA 69.9, POPE 85.7, POPEv2 86.3 — on par with InstructBLIP / Qwen-VL families at much lower compute.
- **WorldPrediction-WM (inverse dynamics — pick the action video clip explaining a state transition):** new SoTA, BASE 63.9% / SFT 65.7%, beating GPT-4o, Claude-3.5, Gemini-2 and larger VLMs.
- **Action anticipation / forecasting:** EK100 beats V-JEPA2 ViT-L-256px at all horizons (and at t=10s: 11.7 vs 8.2 even of ViT-g-384px); COIN next-step new SoTA 56.2%.
- **Selective decoding:** adaptive selection Pareto-dominates uniform; 0.35 Hz matches uniform 1 Hz, **~2.85x** fewer decode ops; average pooling denoises and helps both.
- **Y-Encoder quality:** the JEPA-trained Y-Encoder is more robust to text hard negatives — VL-JEPA_BASE 63.9% SugarCrepe++ / 42.9% VISLA, beating PE-Core and SigLIP2.
- **Ablations:** pretraining is critical (-21.7 classif / -17.3 retrieval without it); Y-Encoder LR multiplier sweet spot 0.05-0.10 (frozen=0.00 costs -7.3 classif); **InfoNCE > cosine/L1/L2** and is the only loss with anti-collapse + unfrozen Y-Encoder support; more predictor layers help (esp. VQA); bidirectional attention and Llama-3 init both help VQA; larger / vision-aligned (PE) text encoders as Y-Encoder boost classification/retrieval.

## Relevance to the EB-JEPA hackathon
This is the JEPA-as-multimodal-prediction track. It maps cleanly onto the eb_jepa recipe: **encoder** = frozen V-JEPA 2 ViT-L X-Encoder (a frozen-backbone world-model-style setup), **predictor** = query-conditioned Llama predictor producing Ŝ_Y (directly analogous to eb_jepa's action/condition-conditioned predictor, here conditioned on a text query instead of an action), **target** = a trainable text Y-Encoder rather than an EMA image target. Crucially for the regularizer axis, it is a **two-view / contrastive (InfoNCE)** instantiation rather than masking + EMA/stop-grad: anti-collapse comes from InfoNCE's uniformity term, and the paper explicitly flags **VICReg and SIGReg as drop-in replacements left to future work** — a concrete, ready-made experiment for an eb_jepa team that already has SIGReg/VC regularizers. It also generalizes the "predict a representation conditioned on a control input" pattern beyond mazes/robotics that eb_jepa world-model work has targeted, and its WorldPrediction-WM inverse-dynamics result is directly relevant to action-conditioned JEPA evaluation. Note the user (Basile Terver) is in the acknowledgments and is cited (Terver et al. 2025; Baldassarre et al. 2025) in the world-modeling related work.

## Caveats / open threads
- **InfoNCE only**: the headline anti-collapse / objective choice is InfoNCE; VICReg/SIGReg, EMA target, and frozen-target variants are asserted compatible but **not run** — the obvious eb_jepa follow-up.
- **No public repo / code or weights stated.**
- Authors disclaim it is *not* a universal VLM replacement: reasoning, tool use, and agentic behaviors (where token-generative VLMs excel) are **not evaluated**.
- Scaling (params + data) showed clear benefits but was **not fully explored**; BASE saw only 3.3-3.6B pairs vs baselines' 40-86B, so some comparisons are not iso-data.
- VQA is evaluated in a **discriminative** (nearest-candidate) setting on perception-heavy benchmarks, not open-ended generative VQA across the board.
- Relatively **weaker on appearance-centric** benchmarks (Kinetics-400, task recognition) — attributed to limited pretraining data.
- Heavy training cost (4 weeks, 24x8 H200) for the BASE model.
