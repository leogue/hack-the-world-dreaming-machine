# AnySat — One Earth-observation model handling many resolutions, scales, and modalities via a multimodal JEPA + scale-adaptive spatial encoder

**Authors:** Guillaume Astruc, Nicolas Gonthier, Clément Mallet, Loïc Landrieu (LASTIG / Univ Gustave Eiffel, IGN, ENSG; CNES; LIGM, ENPC) **Venue/Year:** arXiv preprint, v3 (9 May 2025); cs.CV. Originally Dec 2024 **arXiv:** 2412.14123 **Repo:** https://github.com/gastruc/AnySat (code + models released)

## TL;DR
AnySat is a single Earth-observation (EO) foundation model that trains on *highly heterogeneous* multimodal data without rescaling: 11 sensors, spatial resolutions from 0.2 m to 250 m, tiles from 0.36 to 164K hectares, single images to weekly time series. Two ingredients make this possible: (i) a **scale-adaptive patch encoder** that splits each patch into fixed-size sub-patches so the same network handles any patch size / resolution, and (ii) a **multimodal JEPA** training objective (student/teacher with EMA, latent-space prediction of dropped patches) augmented with a cross-modal contrastive loss. Trained self-supervised on **GeoPlex** (5 datasets, 11 modalities, 249K km², 171B pixels) using 1760 H100 GPU-hours, a *single* pretrained model reaches state-of-the-art after fine-tuning or linear probing on the GeoPlex test sets plus 6 external datasets, across land-cover mapping, tree-species ID, crop classification, change detection, climate-zone classification, and flood/burn-scar/deforestation segmentation. >75% of parameters (73.6% per appendix) are shared across all modalities and resolutions. Claimed as "the first application of JEPA for multimodal EO data."

## Problem & motivation
Unlike natural images (uniform RGB cameras, similar perspectives), EO data varies enormously in modality, scale, spatial/temporal/spectral resolution. Existing EO foundation models are "generally trained on a single dataset with a specific format ... and cannot be applied to datasets with different input types without retraining from scratch — defeating the purpose of foundation models." Prior work added flexibility along one axis (modalities, scale, or spectral bands) but "none fully leverage the diversity of EO sensors," and many still expect a fixed 224×224 input and resize everything (inefficient for small tiles). The key insight exploited as a self-supervision signal: multiple sensors observing the *same area* capture different information but "share the same underlying semantics," so learned representations should be consistent across modalities, and missing modalities should be reconstructable. Because EO data is heavily perturbed by weather, time of day, acquisition angle, the authors argue *latent-space* (feature-predictive) reconstruction is better suited than pixel-space generation — motivating JEPA over MAE-style cross-modal masked autoencoding.

## Method

### Where JEPA is used (vs other SSL)
The core SSL objective **is a JEPA** (the paper adapts Assran et al. I-JEPA [6] to multimodal EO), combined with an auxiliary contrastive loss. There are *two* losses:
- **L_JEPA** (Eq. 5): an L2 distance in embedding space between the student predictor's outputs and the EMA teacher's multimodal embeddings, computed *only on dropped patches*: `L_JEPA = (1/|K|) Σ_{p∈K} ‖f*_{p,pred} − f*_{p,Teacher}‖²₂`. Teacher weights are an EMA of the student (decay 0.996); teacher sees all patches with no masking/dropping; student is discarded after pretraining and the teacher is used downstream. This is the JEPA core: latent-target prediction with stop-gradient EMA target, no pixel decoder. A key advantage stressed: JEPA "eliminates the need for modality-specific decoders," enabling many sensors seamlessly.
- **L_con** (Eq. 2): a modified InfoNCE / OmniSat-style **contrastive loss** on the student patch embeddings, pairing tokens from the same spatial patch but different modalities as positives (temperature τ; cosine similarity). Enforces cross-modal spatial consistency. Ablation shows it mainly helps classification.

So: JEPA = primary feature-predictive objective; contrastive = auxiliary anti-collapse / discriminative regularizer. (Related to the contrastive+feature-predictive combination of data2vec-style methods [10].)

### Scale-adaptive patch encoder φ_patch (the multi-resolution / multi-scale trick)
A tile of S×S m is partitioned into non-overlapping P×P m patches; *all modalities share the same spatial patch layout*. For modality m (resolution R_m, T_m timestamps, C_m channels), a token x^m_p is encoded in three stages: (i) subdivide into fixed-size δ_m×δ_m-pixel sub-patches; (ii) a **modality-specific MLP** φ_proj_m maps each flattened sub-patch to dimension E, with an LTAE collapsing the temporal dimension for time series (T_m>1); (iii) add GSD-based scale-adaptive positional encodings (à la Scale-MAE) and aggregate sub-patches with a **shared** transformer φ_trans (B blocks) via a CLS-like token to one vector f^m_p per modality. Because sub-patch size δ_m is fixed, changing patch size P only changes the *number* of tokens fed to φ_trans, not the architecture — this is what makes one model work across resolutions/scales. δ_m = 1 px for VHR imagery, 10 px for time series.

### Modality-combiner φ_comb
Cross-attention architecture from OmniSat: add absolute positional encoding to each f^m_p, run B self-attention blocks, then cross-attend learned per-patch query tokens against the modality embeddings → one multimodal embedding f*_p per patch.

### Training pipeline (Fig. 3)
Per iteration: randomly pick a GeoPlex dataset d, randomly pick a patch size P from that dataset's allowed set P_d, sample B_d tiles. Then: **patch dropping** (5 random rectangles define dropped set K; all student tokens for p∈K removed — JEPA block masking), **modality & temporal masking** (mask a subset of remaining tokens with a learned f_mask, keeping ≥1 modality per patch; also randomly mask 50% of time-series timestamps), combiner produces f*_{p,S} for kept patches, **predictor** φ_pred (3 self-attn blocks) replaces dropped patches with learned f_drop + positional encoding and predicts their embeddings; teacher encodes everything cleanly; L_JEPA over dropped patches + L_con. Single-pixel/oversized modalities (e.g. MODIS in PLANTED, 250 m > 120 m tile) handled as context tokens (no positional encoding, excluded from L_con).

### Downstream
Classification via a [CLS] token inserted in the combiner cross-attention → linear classifier. Semantic segmentation by concatenating sub-patch embeddings (φ_proj outputs) with patch embeddings (φ_comb outputs) → MLP → δ_m×δ_m×N logits unfolded to pixel resolution (sub-patches give finer predictions). Linear probing attaches a linear layer to the CLS token or dense feature maps. New/unseen sensors: a learned sensor-specific scalar pads missing channels; for entirely new sensors a new projector is randomly initialized and fine-tuned.

### Scale (appendix)
125M params total, 73.6% modality-agnostic & resolution-adaptive: projectors 33M (11 projectors), spatial transformer 45M (3 blocks), combiner 49M (3 self-attn + 1 cross-attn), predictor 29M (single self-attn block). AdamW, LR 5×10⁻⁵, EMA decay 0.996, contrastive temperature 0.1; "all other hyperparameters are shared with original JEPA implementation." 8× H100 for the big datasets.

## Key results
- **Pretraining cost:** "1760 GPU-hours on an NVIDIA H100 GPU." Fine-tuning 10–40 h; linear probing ~2 h on BraDD-S1TS. Inference: 3 ms per TreeSatAI mono-date tile (faster than Scale-MAE's 10 ms, comparable to DOFA 3 ms, OmniSat 2 ms).
- **GeoPlex test sets (single pretrained model):** new SOTA on **TreeSatAI-TS** (+0.9 weighted F1 → 75.1) and **PASTIS-HD** (+2.8 mIoU classification → 72.8 macro F1; +0.2 segmentation → 66.5 mIoU). Near-SOTA on **PLANTED** (61.5 macro F1; ViViT baselines used ~80% more withheld data) and **FLAIR** (55.1 mIoU) despite using only 6.5% of the Sentinel-2 extent UT&T used.
- **External datasets:** "significantly outperforms the state-of-the-art for 6 external datasets," improving **SICKLE** +3.6 mIoU (89.3 FT), **BraDD-S1TS** +10.2 mIoU (80.9 FT), **TimeSen2Crop** +11.0 OA (92.2 FT). A *linearly probed* AnySat (6.1K free params) beats fine-tuned Prithvi2 (630M params) on **Sen1Floods11** (91.1 vs 90.4 mIoU) — "10⁵ fewer free parameters."
- **GeoBench / PANGAEA (Tab. 1):** on par or better than all foundation models (Prithvi, Prithvi2, CROMA, SatMAE, Scale-MAE, DOFA, RemoteCLIP, DINO-S12, SatlasNet) across HLS Burns, Spacenet7, Sen1Floods11, So2Sat, PASTIS-HD. E.g. HLS Burns 90.6 mIoU; PASTIS-HD Fold1 probing 42.7 mIoU.
- **New sensor configs:** SOTA on unseen-band LandSat8 (SICKLE), 9-of-10-band Sentinel-2 (TimeSen2Crop), single-date Sentinel (So2Sat, Sen1Floods11), single-pixel time series. On **HLS Burn Scar** (a sensor *not* in GeoPlex; a new projector is trained) AnySat (90.6 mIoU FT) beats Prithvi (86.9) — note Prithvi was trained on 252M km² of HLS vs GeoPlex's 249K km² with *no* HLS.
- **Ablation (PASTIS-HD Fold 1, Tab. 2):** best config 72.0 F1 / 63.6 mIoU. Random token dropping vs JEPA block masking: 71.3 F1 / 64.1 mIoU (block masking "does not appear to be as critical for EO data than for natural images"). No contrastive: 67.7 F1 (−4.3) / 63.4 mIoU (−0.2) — contrastive mainly helps classification. Naive semseg (no sub-patch features): 61.2 mIoU (−2.4).

## Relevance to the EB-JEPA hackathon
- **Climate / EO spatiotemporal-fields track — direct fit.** AnySat is a JEPA operating exactly on the geospatial multimodal spatiotemporal fields this track targets: optical + radar satellite *time series* over space, plus VHR imagery, with downstream tasks (crop, land cover, flood, burn scar, deforestation, climate-zone) that are squarely environment/climate monitoring. It is a concrete, recent, code-and-models-released demonstration that the EB-JEPA recipe (latent prediction + EMA teacher, here plus a contrastive anti-collapse term) transfers to EO, and a strong baseline / reference architecture for any EO-field JEPA effort.
- **Scale/resolution heterogeneity → core design lesson.** The scale-adaptive patch encoder (fixed sub-patch size, GSD-aware positional encoding) is a reusable trick for *any* spatiotemporal-field model that must ingest variable-resolution grids without resizing — relevant to multi-resolution climate fields.
- **Relation to The Well track.** The Well is a collection of heterogeneous physics/PDE simulation fields (varying grids, channels, time horizons). AnySat's central contribution — *one shared JEPA backbone over heterogeneous multi-resolution, multi-channel, multi-temporal fields with modality-specific input projectors and a scale-adaptive tokenizer* — is structurally the same problem The Well poses (different simulations = different "modalities"/sensors with different field counts and grid spacings). The patch-dropping + temporal-masking JEPA objective and the per-"modality" projector + shared transformer factorization are a candidate template for a unified JEPA across The Well's datasets. Cross-modal contrastive consistency is less obviously transferable (EO's "same area, different sensor" positive-pairing has no direct Well analogue), but the multi-dataset round-robin training loop and scale-adaptive positional encoding do.

## Caveats & open threads
- **Not a pure JEPA:** the contrastive loss is load-bearing for classification (−4.3 F1 without it), so anti-collapse here leans on InfoNCE, not a JEPA-internal variance/covariance (SIGReg/VICReg) mechanism. How much of AnySat's strength is the JEPA vs. the OmniSat-inherited contrastive fusion is only partially disentangled.
- **Block masking barely helps** for EO (random dropping was comparable / slightly better on segmentation); the JEPA spatial-masking inductive bias from natural images does not clearly carry over.
- **Cross-modal supervision needs spatial co-registration:** the method assumes spatially aligned multimodal patches sharing semantics; for FLAIR they discard 93.5% of Sentinel-2 pixels to match VHR extent, and PLANTED has only 1.3M of 2.3M images public — data-availability and alignment caveats matter for reproduction.
- **Per-modality projectors are not shared** (the ~26% non-shared params), and unseen sensors require training a new projector (not usable for probing if too different) — so "any modality" has a fine-tuning cost at the input layer.
- **Appendix has typos / loose reporting** (e.g. contrastive temperature called both τ and γ = 0.1, "Eq. X", scattered dataset-name spellings); some external-dataset rows are FT-only or probing-only, so head-to-head comparisons need care. Hyperparameters are largely deferred to "original JEPA implementation."
- **Single-pretraining-cost number** (1760 H100-h) is reported without a full optimization/throughput breakdown; batch sizes are per-dataset (e.g. PLANTED 2048, PASTIS-HD 8) for memory reasons, complicating direct compute comparison.
